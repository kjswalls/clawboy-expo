## 4. Checks

### Locale Key Completeness

- [ ] List all locale files under `src/i18n/locales/`
- [ ] For each non-English locale: count keys present vs English (`en/common.json`)
- [ ] Flag locales with missing keys as `med` severity (app will show English fallback — not a crash, but bad UX)
- [ ] Verify `i18next` is configured with the correct fallback language

### Hard-Coded String Scan

- [ ] `rg '"[A-Z][a-z]{3,}"' src/components/ app/` — find string literals that look like user-visible copy
- [ ] For each hit: is it already covered by a `t()` call or is it a hard-coded string? Flag hard-coded user-visible strings
- [ ] Focus on: button labels, placeholder text, error messages, empty state copy, modal titles
- [ ] Exclude: test files, pure TypeScript string constants for internal use

### Accessibility Audit

For each interactive UI element found, verify:

- [ ] `<Pressable>` / `<TouchableOpacity>` has `accessibilityLabel` and `accessibilityRole="button"` (or appropriate role)
- [ ] `<TextInput>` has `accessibilityLabel`
- [ ] Toggle/switch elements have `accessibilityRole="switch"` and `accessibilityValue={{ text: state ? 'on' : 'off' }}`
- [ ] Modal / Sheet: has `accessibilityViewIsModal={true}` when open
- [ ] Icon-only buttons (no visible text label): must have `accessibilityLabel`
- [ ] List items with action: `accessibilityRole="button"` or `"menuitem"`
- [ ] Section headings: `accessibilityRole="header"`
- [ ] Loading states: `accessibilityLabel` describes loading state

Specific components to audit:
- [ ] Chat header action buttons
- [ ] Send button
- [ ] Session list items and swipe actions
- [ ] Sidebar open/close trigger
- [ ] All settings toggle switches
- [ ] All settings navigation rows
- [ ] Thinking node expand/collapse
- [ ] Tool call card expand/collapse
- [ ] Code block copy button
- [ ] Media embed (image / video)
- [ ] All modal close buttons
- [ ] Onboarding step navigation buttons
- [ ] Sign-in sheet buttons
- [ ] Add server sheet inputs

### Dynamic Type

- [ ] Sample 10+ `Text` components: verify `allowFontScaling` is not explicitly `false` without justification
- [ ] `fontSize` values use named constants from theme (not inline magic numbers) — easier to audit for scale

### RTL Layout

- [ ] `rg "marginLeft\|paddingLeft\|marginRight\|paddingRight" src/components/ app/` — for each hit, assess if it is asymmetric in a way that would break RTL
- [ ] Flag asymmetric margins not paired with RTL-safe equivalents (flag as `nit` unless the component is explicitly LTR-only)
